##### 一、TDD开发流程

1. 编写测试用例
2. 运行测试，测试用例无法通过测试
3. 编写代码，使测试用例通过测试
4. 优化代码，完成开发
5. 重复上述步骤

<!-- more -->

##### 二、TDD(测试驱动开发)和BDD(行为驱动开发)比较

| TDD                                        | BDD                              |
| ------------------------------------------ | -------------------------------- |
| 先写测试再写代码                           | 先写代码再写测试                 |
| 一般结合单元测试使用，是白盒测试           | 一般结合集成测试使用，是黑盒测试 |
| 测试重点在代码                             | 测试重点在UI                     |
| 安全感低，测试用例都过了项目未必能跑的起来 | 安全感高                         |
| 浅复制组件速度快                           | 速度慢                           |


##### 三、什么情况下用什么

- TDD适合测试函数库，进行单元测试
- BDD适合测试业务，进行集成测试
