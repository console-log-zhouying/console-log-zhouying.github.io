#### 一、弱类型的问题

1. 编译时无法发现代码异常，直至执行时发现

   ```js
   const obj;
   
   obj.foo();
   ```

2. 类型不明确有可能是函数的功能发生改变

   ```js
   function sum(a,b){
     return a + b;
   }
   
   sum(100,100);
   sum(100,'100'); //结果变成字符串
   ```

3. 因为弱类型的关系出现对对象索引器的错误用法

   ```js
   const obj = {}
   
   obj[true] = 100
   
   console.log(obj['true'])
   ```


#### 二、强类型的优势

1. 错误更早暴露。编译时可以检查出异常，不用等到运行时

2. 代码更智能，编码更准确

3. 重构更牢靠

   ```js
   const util = {
     // 某一天突然改名字，弱类型不敢轻易改，因为有错误没有办法立即表现出来。强类型一旦对象属性名发生变化，编译会立即报错，甚至有工具可以自动修改
     aaa: ()=>{ 
       console.log('aaa')
     }
   }
   ```

4. 减少不必要的类型判断
